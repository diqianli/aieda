📖 Barrier 机制概述

什么是 Barrier（屏障）？

屏障是一种同步原语，用于确保多个线程在继续执行之前都到达某个同步点。只有当所有参与线程都调用了 pthread_barrier_wait() 后，所有线程才会同时被释放继续执行。

// pthread_barrier 基本用法
pthread_barrier_t barrier;
pthread_barrier_init(&barrier, NULL, num_threads);

// 在每个线程中
do_work();
pthread_barrier_wait(&barrier);  // 等待其他线程
do_more_work();

硬件原子倾向的特殊影响

Barrier 操作具有 全对全 (all-to-all) 的同步特性，使其对硬件原子操作模式极其敏感：

  • Near-Atomic (近端): 在本核 L1 缓存命中时执行，本地完成 (~3-5 cycles)
  • Far-Atomic (远端): 在本核 cache 未命中，需要 CHI 互连到 Home Node (~20-100+ cycles)
  • 关键因素: Cache 行状态（Unique vs Shared）决定了是 near 还是 far
  • Barrier 的本质: 最后到达的线程触发全局计数器更新，通常为 far-atomic

ARM 上的 Barrier 实现

pthread_barrier 在 ARM 系统上使用原子递增和条件变量实现：

barrier_wait:
    ldaddal x2, x3, [x1]    // 原子递增计数器 (Far-Atomic)
    // 最后一个线程负责唤醒其他线程
    cmp x3, x4              // 检查是否是最后一个线程
    b.ne wait_loop
    // 唤醒所有等待者
    wake_all:
    sev                     // 发送事件
    ret
